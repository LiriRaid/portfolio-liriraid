# Sistema de internacionalización (i18n)

Documentación de `src/app/core/i18n/`.

## ¿Qué contiene?

| Archivo | Descripción |
|---------|-------------|
| `i18n.service.ts` | `I18nService` — fuente única del idioma activo y traducciones |
| `i18n.messages.ts` | Diccionarios `es` y `en` con todas las cadenas del portfolio |
| `i18n-storage.ts` | Persistencia (cookie + localStorage) + resolución de idioma inicial en SSR |
| `i18n.initializer.ts` | `provideI18nInitializer()` — registra el environment initializer |
| `prelanguage-toggle.script.ts` | Script síncrono inyectado en `index.html` para sincronizar el toggle de idioma antes del primer paint |
| `index.ts` | Barrel export del área |

## ¿Para qué existe?

Soportar dos idiomas (español por defecto, inglés) en un único bundle sin frameworks externos pesados. El sistema cubre:

- Lectura/escritura de preferencia (cookie + `localStorage`).
- Resolución correcta del idioma en SSR usando el header `cookie` y `accept-language` del request.
- Animación coordinada de cambio de idioma con GSAP (fade + FLIP de iconos y layout).
- Reactividad mediante signals: cualquier consumidor que use `i18n.t(key)` o `i18n.language()` se re-renderiza al cambiar el idioma.

## API pública

```typescript
class I18nService {
  readonly language: WritableSignal<'es' | 'en'>;
  readonly isSpanish: Signal<boolean>;
  readonly isEnglish: Signal<boolean>;

  initialize(): void;                          // llamado por el environment initializer
  setLanguage(language: PortfolioLanguage): void;
  toggleLanguage(): void;
  t(key: string): string;                      // devuelve la cadena traducida o la clave si no existe
}
```

## Estrategia de resolución del idioma inicial

`resolveInitialLanguage()` aplica la siguiente prioridad:

1. **Browser**: cookie `portfolio-language` → `localStorage['portfolio-language']` → `navigator.language`.
2. **Server (SSR)**: cookie del header `cookie` → header `accept-language` → fallback `'es'`.

La cookie (`SameSite=Lax`, vida 1 año, `Secure` cuando aplica) garantiza que SSR pueda renderizar el HTML en el idioma correcto, evitando el flash de cambio de idioma tras hidratar.

## Pre-language script

`prelanguage-toggle.script.ts` se inyecta en `src/index.html` entre los marcadores `<!-- portfolio-prelang:start -->` y `<!-- portfolio-prelang:end -->` (regenerado por `tools/i18n/generate-prelanguage-index.ts` en `pnpm preindex`). Su trabajo es:

- Leer la preferencia (cookie/localStorage/navigator) antes del primer paint.
- Setear `<html lang>` y `data-language` en el documento.
- Sincronizar los `p-toggleswitch` del header con la clase `is-initial-sync` (sin transiciones) para evitar el "flash" del toggle.

Hace lo mismo que el pretheme script pero para el idioma. Ver [theme.md](theme.md) para la analogía.

## Diccionario de mensajes

`I18N_MESSAGES` es un objeto `Record<PortfolioLanguage, Record<string, string>>` plano con claves dot-notation (`hero.title.line1`, `about.github.repos`, ...). Toda cadena visible al usuario en componentes ya internacionalizados pasa por `i18n.t(key)`.

Para agregar una cadena:
1. Añadir la clave en ambos diccionarios (`es` y `en`) en `i18n.messages.ts`.
2. Consumirla en la plantilla con `{{ t('mi.clave') }}` (donde `t` es un wrapper local que llama a `i18nService.t`).

## Animación de cambio de idioma

`setLanguage()` coordina una animación GSAP con dos fases:

1. **Fade out** del texto traducible actual (selectores `animatedTextSelector`).
2. **FLIP** de iconos e elementos de layout que cambian de posición tras la nueva traducción.
3. **Fade in** del texto nuevo.

Si `prefers-reduced-motion` está activo o no estamos en browser, hace el cambio directo sin animación. Selectores excluidos (carruseles, badges, etc.) viven en propiedades como `animatedTextExcludedClosestSelectors` para evitar animar contenido fijo (nombres de proyecto, tags, etc.).

## Relaciones

- **`app.config.ts`**: registra `provideI18nInitializer()` en los providers.
- **`Header`**: usa `I18nService.t()` y `PortfolioLanguageToggle` (componente que delega en `toggleLanguage()`).
- **Todas las secciones**: usan `i18nService.t(key)` para sus textos.
- **`@shared/utils/gsap-loader`**: carga GSAP en lazy para la animación de cambio.

## Decisiones

- **Sin librerías de i18n externas**: el portfolio tiene textos finitos y dos idiomas; un diccionario JSON-like es suficiente y elimina ~30KB de dependencia.
- **Cookie + localStorage juntos**: la cookie permite leer en SSR; localStorage da resiliencia a borrado de cookies.
- **Animación opt-out por reduced-motion**: respeta accesibilidad sin código adicional por consumidor.
